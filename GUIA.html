Nginx + PHP + MySQL
Sistema de Login

Passo a passo para instalar o stack LEMP e subir um site completo de cadastro e login em PHP.

Índice

  1. Atualizar o sistema

1

Atualizar o sistema

Antes de qualquer coisa, certifique-se que o sistema está atualizado.

Terminal (bash)
$ sudo apt update && sudo apt upgrade -y
2

Instalar o Nginx

Instale o Nginx e habilite-o para iniciar automaticamente com o sistema.

Terminal (bash)
$ sudo apt install nginx -y
$ sudo systemctl enable --now nginx

Verifique se está rodando:

Terminal (bash)
$ sudo systemctl status nginx
✅ Acesse http://IP-do-servidor no navegador — você deve ver a página padrão do Nginx.
3

Instalar o PHP

Instale o PHP-FPM e as extensões necessárias (PDO, MySQL, mbstring etc.).

Terminal (bash)
$ sudo apt install php-fpm php-mysql php-mbstring php-curl php-json -y

Inicie e habilite o serviço (substitua 8.4 pela versão instalada):

Terminal (bash)
$ sudo systemctl enable --now php8.4-fpm
# Descubra a versão exata com:
$ php -v
⚠️ O socket do PHP-FPM fica em /run/php/php-fpm.sock. Se a sua versão do PHP for diferente de 8.4, ajuste o nome do serviço e o socket no arquivo de configuração do Nginx (passo 7).
4

Instalar o MySQL / MariaDB

Instale o MariaDB (100% compatível com MySQL) e execute o script de segurança inicial.

Terminal (bash)
$ sudo apt install mariadb-server -y
$ sudo systemctl enable --now mariadb
$ sudo mariadb-secure-installation
ℹ️ No mariadb-secure-installation: pressione Enter para a senha root (se vazia), responda Y para todas as perguntas de segurança.
5

Criar o banco de dados

Conecte ao MariaDB como root e execute o script SQL do projeto.

Terminal (bash) — entrar no MariaDB
$ sudo mysql

Dentro do prompt do MySQL, cole o conteúdo do arquivo sql/banco.sql (ou rode diretamente):

SQL — banco.sql
CREATE DATABASE IF NOT EXISTS sistema_login
    CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE sistema_login;

CREATE TABLE IF NOT EXISTS usuarios (
    id         INT UNSIGNED  NOT NULL AUTO_INCREMENT,
    nome       VARCHAR(100)  NOT NULL,
    email      VARCHAR(150)  NOT NULL,
    senha      VARCHAR(255)  NOT NULL,
    criado_em  DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uq_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Usuário com acesso apenas a este banco
CREATE USER IF NOT EXISTS 'appuser'@'localhost'
    IDENTIFIED BY 'SenhaForte123!';

GRANT SELECT, INSERT, UPDATE, DELETE
    ON sistema_login.* TO 'appuser'@'localhost';

FLUSH PRIVILEGES;

Ou rode o arquivo diretamente pelo terminal (após copiar para o servidor):

Terminal (bash)
$ sudo mysql < /tmp/banco.sql
⚠️ Lembre-se de alterar 'SenhaForte123!' para uma senha segura tanto no SQL quanto no arquivo config.php.
6

Estrutura dos arquivos do site

Estes são todos os arquivos que compõem o sistema. Cada um está incluído no pacote que você baixou:

projeto-login/ ├── public/ ← raiz do site (servida pelo Nginx) │ ├── index.html ← página principal (formulário login/cadastro) │ ├── config.php ← configuração do banco de dados │ ├── cadastrar.php ← endpoint de cadastro (retorna JSON) │ ├── login.php ← endpoint de login (retorna JSON) │ ├── logout.php ← destroi sessão e redireciona │ └── painel.php ← página protegida (só acessa autenticado) ├── sql/ │ └── banco.sql ← script de criação do banco e tabelas └── nginx-conf/ └── sistema-login ← config do virtual host Nginx
ℹ️ Antes de subir para o servidor, edite public/config.php e ajuste a senha (DB_PASS) para a mesma que você usou no banco.sql.
7

Configurar o Nginx para o site

Remova o virtual host padrão e crie um novo para o sistema de login.

Terminal (bash)
# Remove o virtual host padrão
$ sudo rm /etc/nginx/sites-enabled/default

# Copia o arquivo de configuração do projeto
$ sudo cp /tmp/projeto-login/nginx-conf/sistema-login \
         /etc/nginx/sites-available/sistema-login

# Ativa o virtual host com um symlink
$ sudo ln -s /etc/nginx/sites-available/sistema-login \
              /etc/nginx/sites-enabled/sistema-login

O conteúdo do arquivo de configuração do Nginx é:

nginx — sistema-login
server {
    listen 80 default_server;
    listen [::]:80 default_server;

    root /var/www/sistema-login;
    index index.html index.php;
    server_name _;

    location / {
        try_files $uri $uri/ =404;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_pass unix:/run/php/php-fpm.sock;
    }

    # Bloqueia acesso direto ao config.php
    location = /config.php {
        deny all;
        return 403;
    }

    location ~ /\. { deny all; }
}

Teste a configuração e recarregue o Nginx:

Terminal (bash)
$ sudo nginx -t
$ sudo systemctl reload nginx
8

Copiar os arquivos e definir permissões

Crie o diretório de destino, copie os arquivos PHP/HTML e ajuste as permissões para o usuário do Nginx (www-data).

Terminal (bash)
# Cria o diretório raiz do site
$ sudo mkdir -p /var/www/sistema-login

# Copia os arquivos da pasta public/ para o servidor
$ sudo cp -r /tmp/projeto-login/public/* /var/www/sistema-login/

# Ajusta dono e permissões
$ sudo chown -R www-data:www-data /var/www/sistema-login
$ sudo find /var/www/sistema-login -type f -exec chmod 640 {} \;
$ sudo find /var/www/sistema-login -type d -exec chmod 750 {} \;
✅
Se preferir, você pode transferir os arquivos via SFTP (ex.: FileZilla, WinSCP) direto para /var/www/sistema-login/ e depois rodar apenas o chown.
9

Testar o sistema

Tudo pronto! Abra o navegador e acesse http://IP-do-servidor.

🚀
Fluxo esperado:
1. Acesse a página → formulário de Login/Cadastro
2. Clique em Cadastrar, preencha nome, e-mail e senha
3. Após o cadastro, faça login com as mesmas credenciais
4. Você será redirecionado para o Painel com seus dados
5. Clique em Sair da conta para encerrar a sessão

Se algo não funcionar, verifique os logs:

Terminal (bash)
# Logs do Nginx
$ sudo tail -f /var/log/nginx/error.log

# Status dos serviços
$ sudo systemctl status nginx php8.4-fpm mariadb
⚠️ Em produção: use HTTPS (Certbot/Let's Encrypt), mova o config.php para fora do webroot, e troque as senhas de exemplo por senhas fortes e únicas.
